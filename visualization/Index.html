The Minimax algorithm

Minimax is an adversarial game tree search. X (the Maximizer) wants score +1. O (the Minimizer) wants score −1. Each player assumes the other plays perfectly. At each position, Minimax returns the optimal score from the current player's perspective.

BF Brute Force

Recursively explores every possible game state. No memory — if the same board appears 10 times through different move sequences, it recomputes the full subtree 10 times.

Time: O(bd)   Space: O(d)

DP Memoized

Identical logic but checks a transposition table first. Cache key = board string + "|" + depth. If seen before → return instantly (O(1)). Otherwise recurse and store.

Time: O(U·b)   Space: O(U)

Why DP and not Greedy or Divide & Conquer?

Greedy — fails

Only looks at the current state. No lookahead, no opponent model. Can be trapped by a sequence of moves that looks good immediately but leads to a forced loss. Correctness is non-negotiable.

Divide & Conquer — fails

Works only when subproblems are independent. In Minimax, different move sequences reach the same board state — overlapping subproblems. D&C has no memory, so it recomputes fully every time. That is exactly what Brute Force does.

The overlapping subproblems argument

Consider a 3×3 board. Two different move sequences:

Sequence A

X→(0,0)   O→(1,0)   X→(0,2)

Sequence B

X→(0,2)   O→(1,0)   X→(0,0)

Both arrive at the identical board. Brute Force computes the full subtree twice. DP computes it once, stores the result with key X_XO_____| depth, and returns the cached score instantly on the second visit. At depth 8 on a 5×5 board, 91% of all recursive calls are cache hits.
